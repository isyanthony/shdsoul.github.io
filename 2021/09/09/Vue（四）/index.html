Vue(四):制作Web端音乐播放器

发表于 2021-09-09 分类于 Vue
本文字数： 16k 阅读时长 ≈ 15 分钟
本文用于记录使用Vue制作Web端音乐播放的过程以及一些坑.

效果图

功能一览：

  • 支持单曲循环、随机播放、顺序循环
  • 支持上一首、下一首切换
  • 支持海报处点击暂停
  • 支持拖动进度条条件当前时间
  • 支持音量条的拖动改变音量
  • 支持列表中点击切换相对应的歌曲

<audio/>标签

播放音频需要使用特定的 html标签，例如video、audio标签，这里只讨论使用audio标签，因为我用的就是audio标签.这里也给出相关的标签文档，MDN Audio 标签文档[链接](

属性

这里只列举比较常用的属性

属性名 类型 说明
autoplay boolean 是否开启自动播放
controls 声明了该属性，浏览器将提供一个包含声音，播放进度，播放暂停的控制面板，让用户可以控制音频的播放
currentTime double 读取 currentTime 属性将返回一个双精度浮点值，用以标明以秒为单位的当前音频的播放位置
duration double 音频在时间轴中的持续时间（总长度），以秒为单位。
loop boolean 循环播放音频
muted boolean 是否静音
preload none | metadata | auto 预加载模式，可以加载音频长度或者完整内容
src string 音频地址，可以是http地址
volume int 声音大小，最高为 100

事件

事件名 描述
play 播放开始
pause 播放停止
timeupdate 当 currentTime 时间更新
ended 播放结束时
volumechange 声音改变时

JavaScript

单曲循环

注意到 audio具有loop属性，可以通过布尔值进行设置，并且在ended的事件中判断当前状态是否为单曲循环，从而确定是否切换至下一首，如何接触单曲循环呢？第一种方式，切换播放模式，第二种方式，点击列表中的歌曲切换歌(这个地方其实有点不合理，但是为了偷懒hhh)

随机播放

这里使用了随机打乱数组的算法，将播放列表打乱，然后顺序播放，这也是所谓的伪随机播放，事实上，市面上所有音乐播放器的随机播放都是伪随机，毕竟真真的随机是极有可能会出现重复播放的，然而用户体验的话这样不是很好.

顺序播放

由于，列表歌单和播放歌单我采用了俩个变量进行存储，于是只需要将列表歌单和播放歌单进行合并即可，我这里使用了Object.assign这个方法进行数组合并

时间条

使用了事件 timeupdate，以及属性currentTime、duration，当currentTime，同时更新进度条，所绑定的值 curr，curr = (currentTime / duration ) * 100，对于时间条拖动/点击，则通过 curr反解出 currentTime重新赋值给属性值即可，完成效果.

音量调节

原理和时间条的原理差不多，几乎是一样的，只需要检测值的拖动变化就可以了.

双击歌单播放

如何判断双击？使用计数器进行记录，再加上定时器，在一定时间后清楚计数器即可，这个时间不能太长，我这里采用的500ms，体验下来感觉还不错.
