Electron(二):打包中遇到的坑

发表于 2021-09-09 分类于 Electron
本文字数： 1k 阅读时长 ≈ 1 分钟
本文用于记录使用Electron打包应用为exe时遇到的坑QAQ.

说实话，electron使用过程中坑还真是比较多，搞了大半天终于…终于打包成功了，实在不容易啊，这次就来记录一下，免得以后再遇到不会弄.

打包工具

对于electron应用，有两种打包工具，一种是electron-packager ，一种是electron-builder.

第一种: 打包后会直接生成可执行文件，运行过后就直接是electron应用

第二种: 打包后会生成electoron的exe安装包，运行过后可以安装electron应用即可运行

注意：如上两种都尝试了一下，感觉还是第二种比较使用，毕竟可以很方便的分发给别人使用.

坑

  • can't resolve node moudle：xxx

这种一般都是没有安装某个包，npm install xxx 即可

  • UnhandledPromiseRejectionWarning: ReferenceError: Multispinner is not defined

这种情况一般是在npm run build进行打包时会遇到的，打包时候需要Multispinner，而electron-vue中没有这种插件，需要手动安装.

然后在 build.js 中引入即可

  • Get https://github.com/electron-userland/electron-builder-binaries/releases/xxxxx. dial tcp 20.205.243.166:443: connectex: A connection attempt failed

这种情况就是从Github上下载东西下载失败导致的，于是只能手动去下载，只需复制提示的这个链接去下载相应的东西即可.一般来讲需要下三个东西，winCodeSign 、nsis、nsis-resources

选好相应的版本过后，在C盘中C:\Users\user\AppData\Local\electron-builder\Cache目录下，新建winCodeSign、nsis文件夹，在winCodeSIgn中直接放入解压的所有文件，包括最外层的文件夹一起复制进入这个文件夹即可，而对于nsis需要新建nsis-版本号，nsis-resources-版本号 ，这俩个文件夹只需要复制解压后文件里面的内容，不需要最外层的文件夹.
