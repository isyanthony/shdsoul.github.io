Node.js(一):同步与异步执行流程

发表于 2021-09-09 分类于 Node.js
本文字数： 1.4k 阅读时长 ≈ 1 分钟
本文用于记录JavaScript只能同步与异步的执行流程.

Node.js是JavaScript的一个拓展，使得JavaScript可以用做服务端的语言来写后端，Node.js大部分是由C++编写，Node.js程序运行于单个线程之中，不存在众多服务端语言的多线程等问题，并且Node.js配备了异步I/O功能，其库也使用了非阻塞的方式进行编写，使得Node.js处理并发连接十分轻松，无需引入管理线程并发的负担，当然了并发过于大，也是会造成Bug的。

JavaScript异步与同步

众所周知，JavaScript为单线程的语言，也就是说，不存在多线程，也就不存在程序并行运行，代码执行必然存在先后的问题，这就是这篇文章想要讲述的问题.

JavaScript代码执行流程

先明确一些概念，JavaScript的代码执行是依托于栈的，对于不存在异步的代码，即按照栈先入后出的规则进行执行，然而，对于存在异步操作的代码，就会将异步操作放入一个异步任务执行栈，异步任务又分为宏任务，微任务，等待主执行栈执行完毕后会执行异步任务执行栈，然后依次执行.

宏任务(包括)：

  • script(整体代码)
  • setTimeout, setInterval, setImmediate,
  • I/O
  • UI rendering
    ajax请求不属于宏任务，js线程遇到ajax请求，会将请求交给对应的http线程处理，一旦请求返回结果，就会将对应的回调放入宏任务队列，等请求完成执行。

微任务(包括)：

  • process.nextTick
  • Promise
  • Object.observe(已废弃)
  • MutationObserver(html5新特性)

  • 主执行栈执行完毕后，开始执行异步任务**(注意:**主执行栈执行任务也是需要花费时间的，这也就意味着 setTimeout设置的等待时间只是最少的等待时间，实际时间应该是大于等于这个时间的)
