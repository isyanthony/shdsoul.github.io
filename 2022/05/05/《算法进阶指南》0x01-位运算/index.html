模运算

基本概念

  • 同余式：正整数 a , b 对 p 取模，它们的余数相同，记做 a ≡ b % p或者 a ≡ b (mod p) .
  • n % p 得到结果的正负由被除数n决定,与 p 无关.

运算规则

  • 加法：(a + b) % p = (a % p + b % p) % p

  • 减法：(a - b) % p = (a % p - b % p ) % p

  • 乘法：(a * b) % p = (a % p * b % p) % p

  • 乘方：(a ^ b) % p = ((a % p) ^ b) % p

  • 结合律：

    • ((a + b) % p + c) % p = (a + (b + c) % p) % p
    • ((a * b) % p * c) % p = (a * (b * c) % p) % p

  • 交换律：

    • (a + b) % p = (b + a) % p
    • (a * b) % p = (b * a) % p

  • 分配律：

    • ((a + b) % p * c) % p = ((a * c) % p + (b * c) % p) % p

  • 重要定理：

    • 若a ≡ b (% p)，则对于任意的 c，都有 (a + c) ≡ (b + c) (% p).
    • 若a ≡ b (% p)，则对于任意的正整数 c，都有 (a * c) ≡ (b * c) (% p).
    • 若a ≡ b (% p)，c ≡ d (% p)，则 (a + c) ≡ (b + d) (% p)，(a - c) ≡ (b - d) (% p)，(a * c) ≡ (b * d) (% p).

位运算

常用操作

操作 表达式
取出整数 n 在二进制表示下的第 k 位 (n >> k) & 1
取出整数 n 在二进制表示下的第 0~k -1 位（后 k 位) n & ((1 << k) - 1)
把整数 n 在二进制表示下的第 k 位取反 n ^ (1 << k)
对整数 n 在二进制表示下的第 k 位赋值1 `n
对整数 n 在二进制表示下的第 k 位赋值0 n & (~(1 << k))

常用方法以及性质

成对变换

  • 当 n 为偶数时 n ^ 1 = n + 1
  • 当 n 为奇数时 n ^ 1 = n - 1

lowbit运算

lowbit(n) 定义为 非负整数 n 在二进制表示下 “最低为的 1 以及其后边的所有 0 构成的数值” ，例如 n = 10 = (1010) ，则 lowbit(n) = (10)，() 此处表示二进制下的表示.

底层原理，首先通过 ~n 将 n 取反，则最低位 1 变为 0 而其后方的 0 全部变为 1 ，再通过 n = n + 1 进位使 最低位 1 恢复，此时其前方的数值与原数值恰好相反，则只需要通过 & 运算即可取出后方的值，则得出相应的表达式如下：

lowbit(n) = n & (~n + 1) = n & -n

快速幂+位运算

$a^b$ 这个运算是很有可能会超出int的取值范围的，从而导致计算错误，于是需要采用一些办法来保证不会超过限制，对于 $a^b mod p$ ，由上述的运算法则可以进一步简化.

(a ^ b) % p = ((a % p) ^ b) % p

很快，就会发现纵使这样，也还是会很大概率超出取值范围，于是我们仍需要想办法将这个运算拆成更小粒度的运算，局部处理从而达到效果，于是考虑到任何一个数都是可以由多项式 ci * 2^(i-1) 求和得到 ，ci 为一个数二进制表示中的第 i 位，考虑到求模运算的性质，将 b(假设 b 的二进制表达式中有 k 个比特位 )转化为多项式表达

b = ck * 2^(k-1) + ... + c1 * 2^(0)

于是表达式可以继续通过性质进行化简

a ^ b

= a ^ (ck * 2^(k-1) + ... + c1 * 2^(0))

= a ^ (ck * 2^(k-1)) * ··· * a ^ (c1 * 2^(0))

此时使用模运算中的乘法运算法则进行化简

(a * b) % p = (a % p * b % p) % p

化简如下：

a ^ b

= a ^ (ck * 2^(k-1)) * ··· * a ^ (c1 * 2^(0))

= (a ^ (ck * 2^(k-1))) % p * ··· * (a ^ (c1 * 2^(0)) % p) % p

由于ci为二进制位，只有0 / 1 故 当 b 中二进制位为 0 直接跳过即可，为 1 时进行累乘即可.

于是问题来到了，如何运算 a ^ (2^(k-1)) 如 2 ^ 2 、2 ^ 4 、2 ^ 8、2 ^ 16，每一位二进制位都要进行乘方处理，并且要取得 b的二进制位，对于运算 ，可以采用 a = a * a 进行运算 ， 而取得 b 的二进制位可以采用 移位运算符 >> 以及 & 来进行判断，通过 b & 1 可以判断 b 的最低位二进制位的值，从而可以取出此值，然后再通过 b >>= 1 将最低位舍弃将后方的二进制位进行前移，直至 b 为 0 为止.

D：最短Hamilton路径

原题链接

分析

此题的含义十分清晰，从初始点到终点，求一条路径使花费最小，最简单的想法就求中间路程的全排列从而计算最小值，当点达到一定的数量级，其时间复杂度过大，无法完成计算，于是采用一种更为奇妙的存储状态的方式，利用二进制位来表示，例如 1 << 20 表示将 1 向左移动 20 位，即在 1 的右边添加 20 个 0 ，从最低的 0 开始就表示第一个点的状态，相当于此题中的起点，这样，通过判断二进制位从而判断点是否走过，一下就把状态进行了压缩，那么如何计算最小花费呢？

选择开一个二维数组进行记录，即 f[i][j] ，i 利用其二进制表示当前已走路线的状态，告诉我们哪些已经走过，哪些没有走过 ， j 表示当前走到那里了 ，f[i][j] 在当前状态下以 j 为终点的最小花费.

由于此题中规定了起始点，则有 f[1][0] = 0，1 的二进制表示为 0000001 ，最低位为 1 表示第一个点已经走过，其余的点都没有走过，于是为了方便计算将 f 的其它点均初始化为一个较大的值(这个值并无其它意义，仅仅只是为了方便进行比大小).

则毋庸置疑，所有方案均会以第一个点作为起点开始进行计算，因为其最小.
