React Hook

React Hook的出现是为了解决让组件在函数式编写的时候可以同样使用到Class编写模式下可以使用的state值以及生命周期函数，让React组件开发变的更为容易简单，告别Class编写的冗杂，啰嗦，以及逻辑无法很好复用的缺点.

Hook 使用规则

Hook 就是 JavaScript 函数，但是使用它们会有两个额外的规则：

  • 只能在函数最外层调用 Hook。不要在循环、条件判断或者子函数中调用。
  • 只能在 React的函数组件中调用 Hook。不要在其他 JavaScript 函数中调用。

使用class编写组件的时候，其中有类似于Vue的生命周期函数(只是因为的先学习的Vue而已)，而useEffect的作用就是让函数式组件也有对应其相应的功能，以下是官方文档给与的描述.

useEffect Hook 看做 componentDidMount，componentDidUpdate 和 componentWillUnmount 这三个函数的组合。

官方将Effect称作副作用，简而言之，函数式组件中，state充当了状态层，而JSX充当了UI，而这个Effect相当于进行额外的动态、静态操作，既可以赋能于state也可以仅仅的记录日志.

无需清除的Effect

官方描述如下：

有时候，我们只想在 React 更新 DOM 之后运行一些额外的代码。比如发送网络请求，手动变更 DOM，记录日志，这些都是常见的无需清除的操作。因为我们在执行完这些操作之后，就可以忽略他们了。

Effect的执行是在render渲染完之后才进行的，这点十分容易验证，只需要在useEffect中执行打印日志，以及在外部也执行日志打印即可知道。常见的场景例如，渲染一个用户列表，当渲染完毕时需要发送网络请求获取数据，则此时就可以在useEffect中获取数据然后进行渲染，同Vue中在mounted中获取数据一样.
